Nodeheim Documentation

Overview

Nodeheim is a network discovery and analysis app for Splunk that provides standardized network scanning, analysis, and comparison capabilities. It integrates with Splunk's search and visualization features while maintaining compatibility with both Free and Enterprise editions.

Standard Actions

discover_hosts

Performs network host discovery using non-privileged scans.

| nodeheim-scan source=direct target="192.168.1.0/24"
Note: This action implements the standard "discover" type and can be used interchangeably with other network discovery tools.

investigate_network

Analyzes network topology and structure.

| nodeheim-scan action=investigate_network scan_id="previous_scan_id"
Note: This action implements the standard "investigate" type for network analysis operations.

compare_networks

Compares network snapshots to identify changes.

| nodeheim-scan action=compare_networks scan_id_1="scan1" scan_id_2="scan2"
Note: This action implements the standard "compare" type for network comparison operations.

Data Types

  • ip: Host IP addresses
  • time: Timestamps in ISO format
  • network_topology: Network structure data
  • network_changes: Network difference data

Interoperability

Nodeheim implements standard Splunk action types to ensure interoperability:

  • discover: Can replace network_discovery, host_discovery, asset_discovery
  • investigate: Can replace network_analysis, topology_analysis
  • compare: Can replace network_diff, topology_comparison

Resource Management

The app includes built-in resource management features:

  • Memory usage monitoring
  • CPU usage tracking
  • Disk space management
  • Automatic cleanup routines

Free Edition Support

Nodeheim is fully compatible with Splunk Free Edition with the following optimizations:

  • Local processing mode
  • Resource usage limits
  • Data caching support
  • Progressive scanning
